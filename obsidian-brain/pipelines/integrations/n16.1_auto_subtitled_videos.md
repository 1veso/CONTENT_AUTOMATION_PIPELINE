# n16.1 — Auto-Subtitled Videos (ElevenLabs → NCA Toolkit)

## What it does
Companion flow to [[n16_narrative_chaining]]. Takes a generated video + ElevenLabs voiceover and burns synchronized subtitles via the NCA Toolkit (whisper-style alignment + style overlay).

## Tools used
- ElevenLabs (voice)
- NCA Toolkit (transcribe + burn subtitles)
- Cloudinary or Box for media handoff

- `NCA_TOOLKIT_*` (endpoint + auth)
- `CLOUDINARY_URL` / `BOX_*`

## `[E]` validation errors — diagnosed 2026-05-25 (Session 14)

Two §E nodes show up in `n8n_validate_workflow` (errorCount) as `"Unknown node type"`:

| Node | type / typeVersion | n8n-mcp says | Reality on host |
|---|---|---|---|
| `[E] ElevenLabs` | `n8n-nodes-elevenlabs-enhanced.elevenLabs` v1 | "Unknown node type" | **Loaded.** Credential `elevenLabsApi` ("ElevenLabs", id `3LrGYbduS5r1Hdqb`) bound. Structurally valid. |
| `[E] File Upload` | `n8n-nodes-piapi.fileUpload` v1 | "Unknown node type" | **Loaded.** No credential / no params — unconfigured template stub. |

**Root cause = n8n-mcp validator false-positive, NOT a host problem.** `n8n_validate_workflow` checks node types against n8n-mcp's *bundled static node DB*, which does not include community packages. Both packages are in fact **loaded on the live n8n** — proven via the credential-type registry (the authoritative, shell-free check):

```
GET /api/v1/credentials/schema/elevenLabsApi → HTTP 200  (field xiApiKey)  ⟹ n8n-nodes-elevenlabs-enhanced loaded
GET /api/v1/credentials/schema/piAPIApi      → HTTP 200  (field apiKey)    ⟹ n8n-nodes-piapi loaded
# controls: httpHeaderAuth → 200 (core), bogusFakeApi123 → 404
```

(Credential schema only resolves for a credential type registered by a **loaded** node package — stronger than `ls node_modules`, which can show a package that isn't actually loaded.)

**Action taken: NONE on canvas (operator decision, leave-as-is + document).** These two entries will permanently remain in the MCP `errorCount` because no canvas edit makes n8n-mcp recognize a community node it doesn't bundle — the only ways to clear them (swap to HTTP Request, or delete) change behavior / damage this parked template and were declined. They are **tooling false-positives, not canvas/host/runtime defects.**
