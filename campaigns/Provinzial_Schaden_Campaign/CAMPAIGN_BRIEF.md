## Strategic angle

Insurance is invisible until something goes wrong. Most people don't think about Hausrat, Kfz, Haftpflicht until the moment a Wasserrohrbruch / Auffahrunfall / kaputtes Handy happens — and at that exact moment, they want one thing: someone calm, human, and competent picking up the phone.

This campaign shows **the moment after** the incident. Not the disaster, not the panic — the hand on the shoulder, the quick text, the form filled out in 2 minutes. **Claim handling as the product**, not the policy.

**Tone rules** (from brand brief, reinforced):
- Human. Warm. Real situations.
- Never fear-mongering. Never "what if your house burns down?!"
- Calm authority — Provinzial as the friend who handles it, not the alarm clock.
- Light irony OK occasionally; sarcasm never.
- "Wir sind da, wenn es darauf ankommt." — the operating principle.

## Copywriting standards

Apply both skills per post:

1. **`provinzial-copy` skill** — produces German-language captions matching the brand tone (calm authority, regional warmth, no hype). Output schema:
   ```
   [1-line situation hook]
   [1-line brand reassurance]
   [CTA — direct and simple, e.g. "Im Schadenfall: Provinzial-App öffnen → Foto rein → fertig"]
   #Provinzial #Schaden
   ```

2. **`SEALCaM` skill** (for video content R34, R61, n19, n21) — provides Scene/Energy/Action/Look/Camera/Mood structure for the visual prompt. Apply per scene before sending to Fal/Higgsfield/Veo.

**Hashtag policy:** max 2 brand hashtags per post (`#Provinzial`, `#Schaden`, optionally `#Versicherung`). No hashtag spam.

**CTA examples (rotate, don't repeat):**
- "Im Schadenfall: Provinzial-App öffnen → Foto → fertig"
- "Schaden passiert? Wir sind da. WhatsApp: 0211-XXX"
- "Mehr zur Hausrat-Versicherung → Link in Bio"
- "Termin mit deinem regionalen Berater → provinzial.de"

**Avoid:**
- "Jetzt sichern!"
- "Bevor es zu spät ist!"
- "Was wäre wenn…?!"
- Anything implying urgency or fear

## End-to-end kickoff — webhooks to fire

All requests are `POST` JSON to the webhook URLs below. Each fires the chain and writes a `Pending` row to `PipelineRequests` (`tblLtTpXwFOpzDX4K`), returns `{webhook_id, pipeline, status: "Pending"}` immediately, then continues into the pipeline body.

**Pre-flight checklist before firing any webhook:**

1. ☐ Operator has bound creds in n8n UI: `airtableTokenApi`, Fal `httpHeaderAuth`, `openAiApi`, `openRouterApi`, `httpHeaderAuth Blotato v2`
2. ☐ R57's `tblnpiwNYF3zJXm9Q` and R61's `tbl1hd8yprLTZia4c` Airtable tables are reachable
3. ☐ KIE → Fal swaps are complete in n19 + n21 sub-workflows (otherwise n19/n21 will fail at runtime)
4. ☐ Sheets → Airtable swap done in n3, n16, n19, n21 (if those are part of this run — n3/n16 not used in Schäden v1)
5. ☐ Cost confirmed with operator: R57 (~$2 for 40 images), R61 (~$10 for 25 videos), R34 (~$15 for 20× 3-scene Veo3-fast), n19 (~$8 for 15 ads), n21 (~$15 for 20 UGCs). **Total estimate: ~$50.**
6. ☐ Blotato accounts (TikTok, Meta, Instagram) connected and within posting hold rules — Blotato is **schedule-only**, every call must include a future `scheduledTime`.

**Webhook fire order (parallel-safe):**

```bash
# R57 — 40 static images
curl -X POST 'https://ops.getautomata.ai/webhook/r57' \
  -H 'Content-Type: application/json' \
  -d '{"niche": "Schaeden Alltag", "count": 40, "campaign_id": "Schaden_2026_05"}'

# R61 — 25 cinematic videos (per-record fire; loop client-side or call once with idx range)
for idx in $(seq 1 25); do
  curl -X POST 'https://ops.getautomata.ai/webhook/r61' \
    -H 'Content-Type: application/json' \
    -d "{\"idx\": $idx, \"pillar\": \"Schaden & Service\", \"campaign_id\": \"Schaden_2026_05\"}"
done

# R34 — 20 TikTok 3-scene videos
for i in $(seq 1 20); do
  curl -X POST 'https://ops.getautomata.ai/webhook/r34' \
    -H 'Content-Type: application/json' \
    -d "{\"niche\": \"Schaeden Stories\", \"platform\": \"TikTok\", \"campaign_id\": \"Schaden_2026_05\"}"
done

# n19 — 15 product video ads (requires per-product image url)
# Drive these from a CSV of (product_image_url, platform) pairs.
for url in $(cat schaden_product_images.csv); do
  curl -X POST 'https://ops.getautomata.ai/webhook/n19' \
    -H 'Content-Type: application/json' \
    -d "{\"product_image_url\": \"$url\", \"platform\": \"TikTok\", \"campaign_id\": \"Schaden_2026_05\"}"
done

# n21 — 20 UGC monologues
for char in $(seq 1 20); do
  curl -X POST 'https://ops.getautomata.ai/webhook/n21' \
    -H 'Content-Type: application/json' \
    -d "{\"product\": \"Provinzial Hausrat\", \"character\": \"persona_$char\", \"campaign_id\": \"Schaden_2026_05\"}"
done
```

**Tracking:** every fired webhook writes a `Pending` row to `PipelineRequests`. Operator queries `tblLtTpXwFOpzDX4K` filtered by `pipeline_id IN (R57, R61, R34, n19, n21)` to monitor progress.

**Quality gate:** all 60 video posts (R34 + R61 + n19 + n21 = 80 — note: gate applies to videos only, not R57 statics) must pass the n29 quality gate (Gemini score ≥ 7/10) before Blotato schedule. R57 statics are visual-checked manually in Airtable per existing R57 workflow.
